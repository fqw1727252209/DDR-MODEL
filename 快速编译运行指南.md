# DDR-MODEL 快速编译与运行指南

本文档记录了如何在本模拟器项目中进行编译、测试和运行的完整流程，方便后续复现。

## 1. 编译环境要求

- **操作系统**：Linux 
- **依赖工具**：`cmake` (推荐 3.x 以上版本), `gcc/g++`, `make`

## 2. 编译指南

在项目主目录（`DDR-MODEL`）下，所有的构建都存放在 `build` 文件夹中，以保证源代码目录的整洁。

请在终端下执行以下命令：

```bash
# 1. 切换到项目编译目录 (确保你已经在 DDR-MODEL 下)
cd build

# 2. 生成 Makefile
# (注：如果之前有过乱码或者旧的错误缓存，推荐先清空缓存再重新生成，即执行: rm -f CMakeCache.txt)
cmake .. 

# 3. 开始多线程编译 (使用 -j4 可以并行加速)
make -j4
```

> **说明**：因为我们已经调整了根目录的 `CMakeLists.txt`（默认开启了 `BUILD_TEST=ON`），所以执行上述 `make` 命令不仅会编译各个子模块库（DMU_COMMON, Controller, CHIPort 等），还会自动编译出供运行的入口文件 `PortTest`，并统一输出到 `build` 文件夹中。

## 3. 运行指南

编译成功达到 100% 后，测试可执行程序直接生成在 `build/` 目录下。

```bash
# 在 build 文件夹下直接运行测试程序
./PortTest
```

### 运行时的路径报错说明
我们在首次运行时，遇到了 SystemC 引擎正常拉起，但终端抛出找不到文件的错误：
```text
Failed to open file: ../../ConfigureFile/3ds_map2.json
```
**原因**：
测试代码 `CHITrafficExample.cpp` 中将加载配置文件的路径写成了 `../../ConfigureFile/...`。当我们在 `build/` 目录运行时，`../..` 会向外跳出两级，也就是跳到了 `0_Proj/` 去找 `ConfigureFile`，自然是找不到的（因为 `ConfigureFile` 就在 `DDR-MODEL/` 里面）。

**彻底解决办法**：
找到读取该配置的代码位置，将其路径稍加修改为：
`../ConfigureFile/3ds_map2.json`
修改完毕后，再在 `build` 文件夹里敲一次 `make` 重新编译即可顺利完成业务逻辑的运算输出！

## 4. 后续开发结构建议

随着模拟器的全盘打通，目前的 `PortTest` 仅适合模块化的验证。您可以参考以下结构扩展：
1. **建立统一入口**：以后在根目录建立一个 `simulator/main.cpp` 并引入到根 `CMakeLists.txt` 里，生成官方的 `ddr_simulator` 程序。
2. **规范化相对路径**：建议项目中的配置读取不要以层级跳跃做硬编码，可以通过在 `CMake` 时注入 `${CMAKE_SOURCE_DIR}` 宏定义给 C++ 代码，通过绝对路径拼接解决各种工作路径引发的问题。
